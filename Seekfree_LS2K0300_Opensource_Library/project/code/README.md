# LS2K0300 智能车移植版（精简版）

这是在“逐飞 LS2K0300 开源库”基础上整理的**精简移植版**。

这次改动的目标很明确：

1. **删除蜂鸣器相关使用**
2. **删除 debug 串口相关使用**
3. **暂时不做识别触发动作**
4. **保留核心主链路：UVC 摄像头 + IPS200 + DRV8701 双电机 + 识别串口接收**
5. **强化注释，便于你后续继续扩展任务逻辑**

## 当前保留的功能

- UVC 摄像头采集 `/dev/video0`
- YUYV 转灰度
- 多扫描线循迹
- 模糊 PD 差速控制
- 左右轮 PID 闭环框架
- 视觉串口识别帧解析
- IPS200 状态显示
- DRV8701 双电机输出

## 当前刻意不做的功能

- 不使用蜂鸣器
- 不使用 debug 串口
- 不根据识别类别触发停车、鸣叫、减速等动作

也就是说：

**识别结果仍然接收、解析、显示，但只作为“状态信息”使用，不参与任务动作控制。**

## 适合现在的调试阶段

这个版本更适合你当前的阶段：

- 先把摄像头、循迹、电机、屏幕这条主链跑通
- 再在此基础上逐步加动作策略
- 需要时再把识别类别映射到具体比赛任务

## 目录说明

- `app_config.hpp`：所有关键宏参数，带详细说明
- `app_types.hpp`：公共结构体、状态变量、工具函数
- `uvc_camera.*`：UVC 摄像头采集与灰度转换
- `vision_uart.*`：识别串口协议解析
- `line_track.*`：多扫描线循迹
- `control_utils.*`：PID 与模糊 PD
- `seekfree_hal.*`：逐飞硬件适配层（电机 + IPS200）
- `main.cpp`：主循环与调度入口

## 已绑定的逐飞接口

### 电机
- `ZF_PWM_MOTOR_1`
- `ZF_GPIO_MOTOR_1`
- `ZF_PWM_MOTOR_2`
- `ZF_GPIO_MOTOR_2`

### 屏幕
- `zf_device_ips200`
- `FB_PATH`

## 你后续最常改的地方

### 1. 硬件参数
优先修改 `app_config.hpp`，不要散改源码常数。

### 2. 识别类别业务逻辑
当前版本没有任务动作。以后要加时，建议直接在 `main.cpp` 中新增“识别决策函数”。

### 3. 真实编码器接口
当前仍保留“根据输出命令估算轮速”的占位写法。
如果你后面接上真实编码器，只需要替换 `simulate_encoder_feedback()`。
