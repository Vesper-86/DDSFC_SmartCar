# 十字与环岛整合说明

## 这次具体借鉴了什么

这次不是把 RunACCM2025 全量搬进来，而是只吸收对你当前车型最关键的一段思路：

- 基础循迹保持差速车现有主链；
- 十字 / 环岛识别借鉴你上传的 `image_track` 中“统计 + 状态机 + 补线”方法；
- 控制侧继续沿用当前工程已有的模糊 PD + 左右轮 PID。

## 为什么这样整合

因为你当前这套车已经具备：

- UVC 相机采图
- 灰度循迹
- 差速控制
- 编码器闭环
- 显示与图传

缺的不是底盘控制，而是图像元素区处理。

所以这版工程只补“图像能过十字和环岛”这件事，避免把桥、障碍、任务区状态机一起塞进来，导致你现场更难调。

## 当前实现方式

1. 从灰度图做 ROI Otsu 阈值估计；
2. 把暗色赛道区域转成白色前景；
3. 用底部长白列做种子列；
4. 自底向上搜左右边界；
5. 统计中段丢边、宽度突变、上下角点；
6. 进入十字 / 环岛状态机后执行补线；
7. 输出补线后的中心线给差速控制。

## 当前定位

这版已经适合你下一步做：

- 实车跑十字
- 实车跑左 / 右环岛
- 现场调阈值与状态保持时间
- 再往上接更复杂任务区
